Adventure Assignment Tutorial

Adventure Board API

User

  • The plus button at the end of the adventure list allows for the creation of a new adventure.
  • An adventure always has a creator, a title, a description, and a start and end date.
  • When creating a new adventure it will always assume it’s a full-month adventure until you set the start and end date.
  • Monthly adventures are highlighted with a purple outline.
      Please do not sign up for monthly adventures if you cannot visit the adventure for the whole month.
  • When creating an adventure one can request players that will be automatically assigned to this adventure. This is helpful if the story requires a specific character to be included.
      Please only use this feature if you know that player can actually join the adventure.
  • An adventure can be edited via the 'More Details' button. (Currently not implemented)
  • Via the arrow buttons other than the current week can be inspected for upcoming adventures.
  • The three medal buttons are used to specify a player's priority for a specific adventure.
    • Only players that set at least one priority will be assigned for adventures that week.
    • Only one adventure can be chosen per priority per week.
  • Once the players are assigned to the adventure (this happens every Tuesday or when triggered by an admin), the player can see the assignments in the adventure list. Your own player card is highlighted.

Admin

  • By pressing on on your name in the top right corner you can access the admin panel, in there you can do the following:
    • Force a refresh of the assignments. This will assign all players that set any priorities to the first available adventure. (These refreshes will also happen every Tuesday at a specific time)
    • Refresh the karma of all players. This will add or subtract karma to the players that were assigned to an adventure.
    • Mark which players did not show up. (Currently not implemented)
    • Manually edit the karma of a specific player. (Currently not implemented)
  • Drag and drop player cards to assign them to a different adventure manually.

Demo Specifics

  • Any user can login via there google account.
  • Promoting an admin is only possible via the SQL interface.
  • Acceleration is disabled making the app quite slow.
